# Product Quality Lessons

> product-quality 에이전트가 테스트 시나리오를 작성할 때 반드시 참조하는 파일입니다.
> 사용자가 발견한 버그 = 이 에이전트가 놓친 시나리오.

## 놓친 버그 기록

### [2026-02-22] 프리뷰 입력 영역 전체 화면 너비 (Bug 2+3)
- **사용자 증상**: 프리뷰 팝업 밖 클릭이 막힘, 좌우 이탈해도 닫히지 않음
- **놓친 시나리오**: "프리뷰 팝업 바깥 영역 클릭 → 다른 앱에 전달됨", "프리뷰에서 좌우로 이동 → 프리뷰 닫힘"
- **근본 원인**: updateInputRegion()이 전체 화면 너비 사용. "히트박스=UI 크기" 원칙 미적용
- **추가된 리그레션 시나리오**: 프리뷰 팝업 바깥 클릭 테스트, 좌우 이탈 닫힘 테스트

### [2026-02-23] 런처(0 인스턴스)에서 앱 실행 시 icon fallback
- **사용자 증상**: 인스턴스 0개 상태에서 클릭으로 앱 실행 → 프리뷰가 icon fallback만 표시
- **놓친 시나리오**: "런처에서 앱 실행 → PipeWire 라이브 썸네일"
- **근본 원인**: retryScreencast() 루프가 그룹 윈도우 경로에만 위치, 단일 윈도우 early return 이후 도달 불가
- **추가된 리그레션 시나리오**: 런처(0 인스턴스)에서 앱 실행 시 PipeWire 라이브 썸네일 (icon fallback 아님)

### [2026-02-22] 새 인스턴스 PipeWire fallback 반복 (Bug 1)
- **사용자 증상**: 같은 앱 2번째 인스턴스가 아이콘 fallback
- **놓친 시나리오**: "프리뷰 열린 상태에서 2번째 인스턴스 → PipeWire 라이브 썸네일"
- **근본 원인**: ScreencastingRequest가 one-shot인데 실패 시 재요청 로직 없음
- **추가된 리그레션 시나리오**: 새 인스턴스 PipeWire 라이브 썸네일 확인 (icon fallback 아님)

## 항상 포함할 리그레션 시나리오

(누적 — 매 테스트 계획에 반드시 포함)

1. 독 아이콘 호버 시 줌이 60fps로 부드럽게 동작
2. 독 자동 숨김이 윈도우 겹침 시 정상 작동
3. 독의 기존 기능(줌, 클릭, 드래그)이 새 기능 추가 후에도 정상 작동
4. 프리뷰 팝업 바깥(좌/우) 클릭 → 다른 앱에 전달됨
5. 프리뷰에서 좌/우 이동 → 팝업 영역 벗어나면 닫힘
6. 새 인스턴스 추가 시 PipeWire 라이브 썸네일 (icon fallback 아님)
7. 런처(0 인스턴스)에서 앱 실행 시 PipeWire 라이브 썸네일 (icon fallback 아님)
